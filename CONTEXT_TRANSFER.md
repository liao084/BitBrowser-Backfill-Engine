## 1. 项目背景与目标

*   **业务背景：** 主管已经完成了常规任务的 RPA 脚本（`131_pdd_lpjy.py`），为我们提供了宝贵的参考。
*   **当前目标：** 我们目前正在着手开发针对“历史数据补采”的独立并发引擎（`backfill_engine.py`）。目标是在**比特浏览器（BitBrowser）**环境内，通过**寄生接管（CDP 端口连接）**的方式，实现历史数据的自动化采集。

## 2. 核心架构与技术选型

*   **核心框架：** `Playwright (Async API)` + `asyncio`。
*   **浏览器连接机制 (Parasitic Mode)：**
    *   通过调用比特浏览器的本地接口 (`http://127.0.0.1:54345/browser/open`)，传入对应浏览器的 `bite_id`，获取该实例动态分配的 `CDP 调试端口`。
    *   使用 `await p.chromium.connect_over_cdp("http://127.0.0.1:端口")` 直接接管已启动的浏览器，避免了打包携带体积庞大的浏览器内核。
*   **任务调度机制：**
    *   使用 `asyncio.Queue` 构建任务队列。将较长的时间跨度，按 `chunk_days=3`（每 3 天）进行切分并入队。
    *   启动多个并发 `worker`（目前 `MAX_CONCURRENT_TABS = 5`），并发消费任务队列。

## 3. 核心业务流程 (Worker Logic)

针对工作台弹窗和任务流程，我们梳理并编写了以下交互逻辑：

1.  **环境清理（关闭旧弹窗）：**
    *   检测并关闭可能存在的“昨日日常任务”详情弹窗。
    *   Locator: `i.el-icon-close >> nth=0`
2.  **进入专属补采任务：**
    *   在工作台大盘中，点击指定的补采任务卡片进入详情。
    *   Locator: `div.workTool_page_card_test_dataCard >> nth=0`
3.  **日期注入：**
    *   使用 Playwright 的 `page.evaluate()` 向前端的 Vue/Element-UI 日期控件直接注入 `start_date` 和 `end_date`，并触发相应的事件。
4.  **物理安全保险 (SAFE_MODE) —— 核心调试机制：**
    *   **当前代码中的全局开关 `SAFE_MODE = True` 处于开启状态。**
    *   在注入日期后，程序将记录日志并**无限期休眠**，**不会**去点击后续的“启动检测”或“补齐”按钮。这专为实机环境的人工安全校验而设计。
5.  **后续检测与补采（关闭 SAFE_MODE 后执行）：**
    *   点击【启动检测】：`#checkbutn`
    *   点击【补齐数据】：`span.lostDataBtn`
    *   点击【全店补齐】：`#loseDays_shop_btn`
